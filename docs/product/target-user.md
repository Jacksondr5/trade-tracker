# Target User

## Purpose

This document defines the target user Trade Tracker is built for.

Use it when product, UX, copy, or navigation decisions depend on who the product should optimize for.

## Summary

Trade Tracker is built for a developing, process-driven discretionary investor-trader.

This person is becoming more active in investing and swing trading and wants a durable system for building, refining, and faithfully executing a personal approach.

## What This User Is Trying To Do

- split capital into distinct buckets and evaluate performance across them
- develop campaigns and trade plans for active swing-trading opportunities
- execute trades faithfully against prepared plans
- stay disciplined and emotionally controlled during live decisions
- preserve enough evidence to run honest retrospectives afterward
- use those retrospectives to refine process, discipline, and strategy over time

## What The Product Should Assume

- the target user is not a total beginner, but is still developing process maturity
- the target user benefits from structure, context, and review discipline
- the target user does not need beginner-style education or heavy hand-holding
- the target user values clarity, density, and speed over mass-market polish

## What The Product Should Protect Against

- memory-driven trading
- entering trades without a clear plan
- drifting from the plan during live execution
- emotional reactions overriding prepared conviction
- losing the reasoning and market context needed for later review
- being unable to tell which capital buckets, strategies, or workflows are actually working

## Product Implication

Trade Tracker should help this user become more disciplined, more reviewable, and more independent over time.
