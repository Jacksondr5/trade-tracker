# Product Docs

This directory contains evergreen product documentation for Trade Tracker.

Unlike the dated documents in `docs/plans/`, these docs are meant to stay current and describe the product as a whole rather than a single feature effort.

## Current docs

- `product-vision.md` - mission, north star, intended outcomes, and product boundaries
- `product-principles.md` - durable product rules for feature, workflow, and UX decisions
- `information-architecture.md` - core objects, relationships, overlays, and intentional flexibility in the model
- `ux-principles.md` - interaction and workflow rules for clarity, context, review, and efficient use
- `navigation-model.md` - global, local, mobile, and direct-jump navigation structure across the product
- `feature-philosophy.md` - what each major feature area is for, what it should optimize for, and what it should avoid becoming
- `visual-design-system.md` - concrete palette, surface, typography, spacing, and workflow-specific visual rules
- `content-and-copy-principles.md` - concrete voice, naming, action, empty-state, helper-text, and feedback-writing rules
- `roadmap.md` - phased ordering of major product work and the current strategic sequence
- `technical-architecture-overview.md` - repo structure, mandatory engineering conventions, and extension guardrails
